## v8.4 — 2026-07-17
> **The third external-review intake** — a full technical/security audit of the published 8.3.0 tarball (20 findings + a 156-case dynamic matrix), processed per the §15 protocol: every load-bearing claim was **reproduced against the shipped code** before any decision, and Claude Code semantics were re-verified against current docs. The review is serious and largely correct; its own headline ("accident guard, not a security boundary") is what we already claim. This release fixes the CONFIRMED real bugs and keeps the language honest; the "make it a security product" asks stay **rejected with reasons** (below).
- **CONFIRMED + FIXED — filesystem containment (the worst findings):** the provenance stamp `.claude/.vibe-playbook`, if pre-created as a **symlink or hardlink**, made `writeFileSync` create/overwrite a file OUTSIDE the target (silent, exit 0 — reproduced 3/3); and a **nested symlinked dir** (`.claude/hooks`→external) let `cpSync` write template files outside the target (reproduced). The CLI now refuses any symlink/hardlink on the write path — target, `target/.claude`, the stamp (symlink + `nlink>1`), and **every destination dir at/below the target**. Ancestors ABOVE the target are deliberately still resolved (normal realpath — e.g. `/tmp`→`/private/tmp`).
- **CONFIRMED + FIXED — GATE 4 wrong-target merge:** a marker `wip/P-1` allowed `git merge wip/P-10 -m wip/P-1` (the approved name was matched as ANY token, not the merge operand — reproduced). main-guard now parses the **actual merge OPERAND** (skipping option-values like `-m`/`--strategy-option`) and requires it to equal the approved branch. *(The marker stays a branch-name drift-guard by design — see the reject list.)*
- **CONFIRMED + FIXED — main-guard robustness (one python/`shlex` parse replaces layered regex):** absolute-path `git` (`/usr/bin/git commit`) is now recognised; a shell operator INSIDE a quoted commit message (`git commit -m "docs: a; b"`) no longer false-blocks as a compound command (TOCTOU protection for real operators intact). The fragile bash/sed string-parsing that produced five bugs across v8–v8.3 (prefix, comment, quoted-message, token-anywhere, absolute-path) is gone.
- **ADOPTED (user decision) — hooks fail-closed on malformed input:** guard-env + main-guard now **block (exit 2) on non-empty unparseable / non-dict hook input** (was silent exit 0) with a clear "update or remove the hooks" message; empty stdin still exits 0. Mirrors the existing "python3 missing → block" choice. Trade-off accepted: a future Claude Code JSON-schema change would block tool calls until the hooks are updated (loud + recoverable).
- **ADOPTED — PreCompact actually blocks on failure:** verified that only **exit 2** blocks compaction in current Claude Code (exit 1 does not), so a snapshot **write failure now exits 2** (was 1 — silent) and a `docs/archive` that resolves outside the project is refused. The comment claiming it "cannot stop compaction" was wrong and is fixed.
- **ADOPTED (user decision) — default design mode is now `none`** (was `sync`): a bare `init` ships NO design surface; UI projects opt in with `--design sync|first`. The Claude Design MCP/trust surface is opt-in, not automatic (review F-15). Existing scaffolds are unaffected (each is stamped). `--` option-terminator + leading-dash targets now supported (F-16).
- **ADOPTED — honest language (our own mandate):** `.env.example` "the AI cannot access it" → "kept out of the agent's reach … an accident guard, not a security boundary"; README "physically enforced" → "guarded by hooks (accident guards …)"; the v8.3 "fully backward compatible" clause narrowed to "clean scaffold; no auto-upgrade". STARTGUIDE high-assurance bullet now names the real hardening: verifier `isolation: worktree`, the session `sandbox` fields, and `.claude/**` under CODEOWNERS.
- **REJECTED (recorded, per protocol — reproduced/considered, then declined):** cryptographic GATE 4 marker (SHA/base/nonce/single-use — the marker is a drift-guard; the hard boundary is branch protection + PR review, already documented) · full migration/upgrade/manifest/rollback CLI (deliberate no-force + collision refusal + stamp; the pinned-version model is the design — a manual three-way merge is the documented upgrade) · executable/hashed state machine + gate transition validator (ceremony; the GATE 2 plan-on-disk + `disable-model-invocation` cover the real gap) · sandbox-on-by-default (session-level only, OS/version-specific, breaks legit flows — documented as the opt-in hard boundary) · exhaustive git-verb blacklist (rebase/pull/reset/switch/update-ref… — a treadmill; those land on branch protection, and settings already `ask` on the destructive ones) · CI OS-matrix / native-Windows (WSL is the documented path). Note: `isolation: worktree` is documented, not defaulted (it changes the verifier's checkout semantics).
